# 功能模块内容重做设计

日期：2026-06-29

## 背景

当前“功能模块”弹窗已经包含平生热力图、今日联动、事件流、模块宫格、关于 App、使用指导、问题反馈和退出登录。问题不是内容太少，而是信息层级偏散：用户点开后不容易立刻判断“今天整体状态如何”和“下一步该进入哪个模块”。

本次只重做“功能模块”弹窗内容。左上角入口按钮、打开方式、模块跳转逻辑保持不变。

## 目标

把“功能模块”做成全局模块中心，让用户点开后能快速完成三件事：

1. 看到今日核心状态。
2. 切换到目标模块。
3. 回看最近几条关键动态。

## 非目标

1. 不重做左上角入口按钮。
2. 不新增业务数据录入能力。
3. 不改变计划、财务、饮食、锻炼、健康各模块的页面结构。
4. 不把功能模块做成完整个人主页。

## 信息结构

弹窗内容按以下顺序组织：

1. 标题区：保留“功能模块”标题和关闭按钮。
2. 今日紧凑摘要：展示待办、饮食热量、锻炼组数三个核心状态。
3. 模块状态宫格：展示计划、财务、饮食、锻炼、健康、设置六个入口。
4. 最近动态：展示最近 3 条关键记录。
5. 辅助入口：关于 App、使用指导、问题反馈、退出登录。

## 今日紧凑摘要

摘要使用一行或一张低高度状态条，不使用大卡片。

默认内容：

- 待办：显示未完成数量，例如“待办 3 项”。
- 饮食：显示今日热量，例如“饮食 1680 kcal”。
- 锻炼：显示今日训练组数，例如“锻炼 4 组”。

当某项没有数据时显示 0，不隐藏该项，避免布局跳动。

## 模块状态宫格

模块入口使用紧凑小卡片，保留现有模块点击行为和测试 key。每个卡片包含图标、模块名和一条状态文案。

模块状态文案：

- 计划：未完成待办数量，例如“3 项待办”。
- 财务：优先显示今日支出；如果当前没有今日支出数据源，则显示本月记录数量或“查看账本”。
- 饮食：今日热量，例如“1680 kcal”。
- 锻炼：今日训练组数，例如“4 组训练”。
- 健康：显示“今日有记录”或“未记录”。
- 设置：显示“账号与偏好”。

当前所在模块使用轻量高亮，避免大面积强色块。非当前模块保持普通背景和边框。

## 最近动态

最近动态只展示 3 条，来源继续使用现有 `events` 数据。每条动态使用一行紧凑列表项，包含类型图标、标题和时间信息。

没有动态时显示空状态文案：“今天还没有新记录”。

## 热力图处理

当前顶部热力图占用空间较大，不再作为功能模块弹窗的主内容。功能模块内不展示完整热力图。

如果需要保留记录感，只保留一个低高度的记录摘要，例如“本月已记录 12 天”。完整热力图后续更适合迁移到健康或个人页面。

## 辅助入口

关于 App、使用指导、问题反馈、退出登录放在底部，使用小号列表样式，降低视觉权重。

退出登录保持红色风险提示样式。

## 交互

1. 点击模块卡片后保持现有模块切换逻辑。
2. 点击设置卡片打开现有设置弹窗。
3. 点击关于、使用指导、问题反馈继续打开现有弹窗。
4. 关闭按钮继续关闭功能模块弹窗。

## 验证

1. Flutter widget 测试通过。
2. 现有 `module_sheet_*` key 仍可找到对应模块入口。
3. 在计划、财务、饮食、锻炼、健康模块中打开功能模块弹窗，当前模块高亮正确。
4. 最近动态为空和非空两种状态显示正常。
5. 弹窗在常见手机宽度下不出现文字重叠或按钮溢出。
